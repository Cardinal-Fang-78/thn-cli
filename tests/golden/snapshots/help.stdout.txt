usage: thn [-h] [--version]
           {new,blueprint,dev,diag,hub,init,keys,list,make,plugins,registry,routing,sync,cli,delta,docs,sync-remote,sync-status,web,tasks,ui} ...

THN Master Control / THN CLI

positional arguments:
  {new,blueprint,dev,diag,hub,init,keys,list,make,plugins,registry,routing,sync,cli,delta,docs,sync-remote,sync-status,web,tasks,ui}
    new                 Create a new THN blueprint.
    blueprint           Blueprint utilities for THN.
    dev                 Developer utilities for THN.
    diag                Diagnostic utilities.
    hub                 THN Hub / Nexus integration commands.
    init                Initialize THN system folders.
    keys                Manage THN Ed25519 signing keys and trusted key store.
    list                List all THN directories.
    make                Create THN projects or modules (Hybrid-Standard).
    plugins             Manage THN plugins.
    registry            Inspect and modify the THN registry.
    routing             Routing utilities for THN.
    sync                Sync V2 envelope utilities.
    cli                 Sync CLI tool assets via Sync V2 routing and executor.
    delta               CDC-delta inspection, manifest building, chunk-store
                        tools.
    docs                Sync documentation assets through Sync V2 routing and
                        executor.
    sync-remote         Push Sync V2 envelopes to a remote THN Sync server.
    sync-status         View and manage THN Sync V2 history.
    web                 Sync web assets through Sync V2 routing and executor.
    tasks               Manage scheduled THN tasks.
    ui                  THN UI subsystem commands.

options:
  -h, --help            show this help message and exit
  --version             show program's version number and exit
